# Лабораторная работа №4 (АОИС)

В этой работе реализована хеш-таблица с открытой адресацией.
Коллизии решаются линейным поиском (linear probing).

## Хеш-функция

```
h(V) = V mod H + B
```

где:

- `V` — числовой ключ;
- `H` — размер таблицы;
- `B` — базовый (начальный) адрес.

`V` вычисляется по первым двум буквам ключевого слова на основе алфавита.
Поддерживаются русский и латинский алфавиты.

## Что реализовано

**CRUD-операции:**
- `create(key, value)` — добавить новую пару ключ-значение;
- `read(key)` — получить значение по ключу;
- `update(key, value)` — обновить значение;
- `delete(key)` — удалить запись.

**Разрешение коллизий:**
- при занятости ячейки выполняется последовательный переход к следующей;
- в конце таблицы поиск продолжается с начала (циклически).

**Обработка типовых ситуаций:**
- запрет дублирования ключей;
- ошибка при переполнении таблицы;
- корректная работа после удалений (через пометки удалённых слотов).

**Просмотр внутреннего состояния:**
- `dump()` в CLI выводит снимок всех слотов таблицы;
- отображаются флаги, `P0`, `V` и `h`.

**Авторасширение:**
- не реализовано (размер фиксированный по требованиям ЛР).

## Краткая теория

**Идея хеш-таблицы**
Ключ преобразуется в адрес ячейки через хеш-функцию. Это даёт быстрый доступ к данным в среднем случае.

**Коллизии**
Коллизия возникает, когда разные ключи дают один и тот же адрес.
В этой работе используется линейное пробирование: если место занято, берётся следующая позиция.

**Почему нужны пометки удалённых элементов**
Если после удаления просто освободить ячейку, можно «сломать» цепочку поиска.
Поэтому удалённые элементы помечаются флажком `D`, чтобы поиск оставался корректным.

## Как использовать

### Пример работы в коде

```go
package main

import "lab4/internal/hashtable"

func main() {
	table, _ := hashtable.NewTable(20)
	table.Insert("Анатомия", "Раздел медицины о строении организма.")
	entry, _ := table.Find("Анатомия")
	_ = table.Update("Анатомия", "Обновлённое значение")
	_ = table.Delete("Анатомия")
	_ = entry
}
```
